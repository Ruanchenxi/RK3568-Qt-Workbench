## 注意事项（踩坑防范）

### 架构
- `m_keySessionId` 是**串口会话编号**，不是钥匙物理身份。同一把钥匙拔插后 session 变。**不能用 session 判断是否同一把钥匙**（C2 搁置原因）。
- `AutoReturnCleanup` 分支的收口逻辑**必须逐字保留**，不得简化或重排。任何看起来"可以优化"的地方都先确认，否则影响已验证的自动回传链路。
- `callTermination` 是异步的，不阻塞 UI。`removeTicket` 在 `callTermination` 之后**同步**执行，不等回调。

### URL 构造
- `apiUrl()` 默认格式：`http://xxx/api/kids-outage/third-api`
- termination 接口：`http://xxx/api/kids-outage/task/termination`
- 构造方式：去掉 `/third-api` 后缀，拼接 `/task/termination`
- Query 参数：`id={taskId}&reason=管理员手工删除钥匙任务`

### Token
- 用 `AuthServiceAdapter` 获取 token，不直接引用 `AuthService` 单例
- Header：`kids-auth: bearer {token}`（小写 bearer）+ `kids-requested-with: KidsHttpRequest`

### 搁置原因
- C2（撤销链 session 校验）：需要协议层提供钥匙 UID/序列号，才能区分"原钥匙"和"当前钥匙"
- C4（反向对账）：依赖 C2 的 `transferSessionId`，同因搁置
