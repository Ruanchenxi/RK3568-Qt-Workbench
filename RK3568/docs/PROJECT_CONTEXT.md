# 项目背景与开发上下文（PROJECT_CONTEXT）

Status: Active  
Owner: 产品/架构共同维护  
Last Updated: 2026-03-11

## 1. 文档目的

本文件用于让“没有历史上下文的人”快速理解：

1. 这个项目要解决什么问题。  
2. 当前做到什么程度。  
3. 近期开发应优先做什么。  
4. 哪些边界不能随意突破。  

## 2. 项目定义（一句话）

RK3568 项目是对既有可运行产品进行功能复刻与工程重构的项目，目标在 PC 与 RK3588 Ubuntu 双端运行，并通过闭环验证逐步收敛到稳定版本。

## 3. 业务目标与工程目标

### 3.1 业务目标

1. 复刻原产品关键流程，确保用户侧行为一致。  
2. 支持钥匙管理、日志查看、系统配置、工作台业务。  
3. 支持多种登录模式（账号密码、后续刷卡、后续指纹）。  

### 3.2 工程目标

1. 将平铺代码收敛到可维护目录结构（Feature-First Lite）。  
2. 降低单人维护成本，提升定位速度与回滚安全性。  
3. 建立可执行门禁与无硬件验证路径。  

## 4. 运行平台与部署场景

1. 开发验证平台：Windows（Qt Creator + qmake）。  
2. 目标部署平台：RK3588 Ubuntu。  
3. 构建方式：当前维持 `qmake + .pri`，不在本阶段切 CMake。  

## 5. 页面与能力现状（2026-03-02）

### 5.1 工作台（Workbench）

1. 主要接口链路已可用。  
2. 后续可能随后端 API 调整进行适配。  

### 5.2 服务日志（Log）

1. 主流程已可用。  
2. 后续重点在筛选、可读性与诊断效率。  

### 5.3 钥匙管理（Key）

1. 传票主链已并入主程序：工作台 JSON -> 本地 HTTP 接收 -> 系统票池 -> 手动/自动传票。  
2. 串口报文页、HTTP 客户端/服务端报文页已承担主链路诊断作用。  
3. 自动回传主链已在本地程序真机环境下验证通过。  
4. 手工 `初始化(INIT 0x02)` 与手工 `下载 RFID(DN_RFID 0x1A)` 已正式接入主程序。  
5. 当前重点已转向：送电样本、真实多帧回传样本、以及后续业务收口。  

### 5.4 系统设置（System）

1. 基础配置已可用。  
2. 用户身份采集模块作为后续刷卡/指纹登录配套能力。  

### 5.5 登录（Auth）

1. 账号密码登录已可用。  
2. 刷卡与指纹能力已预留架构插槽，后续逐步接入。  

### 5.6 虚拟键盘（Keyboard）

1. 后续优先采用 RK3588 Ubuntu 系统键盘能力。  

## 6. 外部依赖与硬件边界

1. 原产品硬件：X5 钥匙底座 + X5 钥匙。  
2. 对照策略：原产品行为对照 -> 新系统闭环复刻。  
3. 串口规划红线：  
   - `/dev/ttyS4`：钥匙协议链路  
   - `/dev/ttyS3`：刷卡采集链路（后续接入）  
4. 登录能力通过外部 API 对接（账号/刷卡/指纹均可能走 API）。  

## 7. 当前技术约束（必须遵守）

1. `.ui` 文件固定在项目根 `ui/`，不迁移。  
2. 协议关键语义（7E6C/CRC/重试/稳定窗）不随意改。  
3. 页面层不得直连底层实现（串口/网络/进程）。  
4. 每步改动必须满足可编译、可运行、可回滚。  
5. 结构调整后必须执行门禁：`.\tools\arch_guard.ps1 -Phase 3`。  

## 8. 当前开发方法论

1. 小步提交：每次改动聚焦单一问题。  
2. 闭环验证：每个链路（例如 SET_COM/Q_TASK/DEL）单独验收。  
3. 可回滚优先：先保证安全，再追求速度。  
4. 文档同步：规则变更时文档与脚本必须同步更新。  

## 9. 近期优先级（建议顺序）

1. 继续收 **送电任务样本** 与 **真实多帧回传样本**。  
2. 保持 `传票 / 手工 INIT / 手工下载 RFID / 自动回传` 主链稳定。  
3. 在主链完成后，再统一处理撤销/删除系统票、重复下发高级规则等业务 bug。  
4. 保持已完成功能稳定，避免结构性回退。  

## 10. 非目标（当前阶段不做）

1. 不进行大规模命名重构。  
2. 不切换 CMake。  
3. 不在未验证前重写协议核心语义。  

## 11. 新 Agent 最小上手路径（执行版）

1. 读 `docs/README.md`（总入口）。  
2. 读本文件（业务与约束背景）。  
3. 读 `architecture/CURRENT_FOLDER_LAYOUT.md`（当前结构）。  
4. 读 `architecture/CODEMAP.md`（修改入口定位）。  
5. 读 `architecture/DEPENDENCY_RULES.md`（依赖红线）。  
6. 若涉及串口协议改动，必须先读：  
   - `serial/SERIAL_PACKET_AGENT_GUIDE.md`（链路架构、状态机、改动流程）  
   - `serial/PROTOCOL_FRAMES_REFERENCE.md`（逐字节帧格式、CRC 向量、常量速查）  
7. 读 `replay/REPLAY_SPEC.md`（无硬件验证）。  
8. 修改后执行 `.\tools\arch_guard.ps1 -Phase 3` 并完成主路径验证。
