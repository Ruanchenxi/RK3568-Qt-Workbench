## 首次部署（只需做一次）

部署完新二进制后，SSH 到板端执行以下步骤：

### 1. 修改 start.sh 加入日志重定向

```bash
sed -i 's|exec ./RK3568|exec ./RK3568 >> "$LOG" 2>\&1|' /home/linaro/Yunwang-Technology/start.sh
```

确认修改正确（最后两行应含 `exec ./RK3568 >> "$LOG"`）：

```bash
tail -5 /home/linaro/Yunwang-Technology/start.sh
```

### 2. 配置 systemd 开机自启

```bash
mkdir -p ~/.config/systemd/user

cat > ~/.config/systemd/user/yunwang.service << 'EOF'
[Unit]
Description=Yunwang Technology App
After=graphical-session.target

[Service]
ExecStart=/home/linaro/Yunwang-Technology/start.sh
Restart=on-failure
RestartSec=3

[Install]
WantedBy=default.target
EOF

loginctl enable-linger linaro
systemctl --user daemon-reload
systemctl --user enable yunwang.service
```

### 3. 启动并验证

```bash
systemctl --user start yunwang.service
systemctl --user status yunwang.service
```

看到 `active (running)` 说明正常。

### 4. 查看日志确认无报错

```bash
tail -f /home/linaro/Yunwang-Technology/logs/stdout.log
```

### 5. 配置桌面图标（可选）

```bash
mkdir -p ~/Desktop

cat > ~/Desktop/yunwang.desktop << 'EOF'
[Desktop Entry]
Type=Application
Name=智能停送电管控系统
Exec=/home/linaro/Yunwang-Technology/start.sh
Icon=utilities-terminal
Terminal=false
Categories=Utility;
EOF

chmod +x ~/Desktop/yunwang.desktop
```

XFCE 桌面右键图标 → 允许执行，之后双击可手动启动（systemd 才是正式自启方式）。

## 日志位置

| 日志 | 路径 | 说明 |
|------|------|------|
| 运行日志 | `logs/stdout.log` | 程序所有输出，超 20MB 自动归档 |
| 归档日志 | `logs/stdout.prev.log` | 上一份超大日志 |
| 应用日志 | `logs/app.log` | 程序自身写入的结构化日志 |
| 票据 JSON | `logs/tickets/` | 传入票据原始数据，回传成功后自动删除 |
| 串口日志 | `logs/serial/` | 手动从钥匙管理页面导出 |

## 板端目录结构

```
/home/linaro/Yunwang-Technology/
├── RK3568          <- 主程序（每次部署更新这个）
├── start.sh        <- 启动脚本（一般不动）
├── config/         <- 配置文件（升级时保留，不要删）
├── data/           <- ticket_store.json（升级时保留，不要删）
└── logs/
    ├── stdout.log
    ├── stdout.prev.log
    ├── app.log
    ├── tickets/    <- 票据 JSON（自动管理）
    └── serial/     <- 串口日志导出
```

**升级时只替换 RK3568 二进制，其余目录全部保留。**
